# GitHub SpecKit: вайб-кодинг на основе спецификаций

## Что это

Инструмент для превращения хаотичных промптов в чёткие структурированные документы для AI-агентов.

## Почему важно

- Уменьшает ошибки генерации кода
- Предоставляет правильный контекст для нейросети
- Минимизирует "фантазии" ИИ

## Ключевые команды

### `/speckit.constitution` — "ДНК проекта"
**Файл:** `.specify/memory/constitution.md`  
Важнейший файл с принципами, от которых AI не должен отступать.

### `/speckit.specify` — Что нужно построить
**Файл:** `specs/001-feature-name/spec.md`  
Здесь описываете **ЧТО** и **ЗАЧЕМ**, игнорируя технические детали.

### Другие команды
- `/speckit.plan` — технические детали
- `/speckit.analyze` — анализ кода
- `/speckit.implement` — реализация

## Best Practices

1. Начните с минимума в constitution
2. Пишите для человека, а не для машины
3. Используйте "ДОЛЖНА/МОЖЕТ/НЕ ДОЛЖНА"
4. Разделяйте MVP и будущие фичи
