# Алгоритмические паттерны для собеседований

## Что это такое

Три фундаментальных алгоритмических паттерна, которые покрывают **~60% задач на технических собеседованиях**:

1. **Two Pointers (Два указателя)** — обход массива/строки двумя индексами в одном цикле
2. **Hashmap (Хеш-таблица)** — структура `ключ → значение` для O(1) поиска и подсчёта
3. **Stack (Стек)** — LIFO-структура для скобок, вложенных структур и отката действий

Вместо интуитивного перебора эти паттерны позволяют **распознавать тип задачи** и применять готовое оптимальное решение.

## Почему это важно

Без знания паттернов разработчики часто:
- Пишут решения с O(n²) там, где возможно O(n)
- Тратят время на изобретение велосипеда
- Не могут объяснить интервьюеру ход решения
- Выбирают неоптимальные подходы под давлением

Знание паттернов даёт:
- **Автоматическое распознавание** типа задачи
- **Оптимальную сложность** O(n) вместо O(n²)
- **Структурированное объяснение** решения интервьюеру
- **Уверенность** на технических собеседованиях

## Ключевые идеи

### Паттерн 1: Two Pointers (Два указателя)

**Суть:** Использовать два указателя для обхода массива или строки в одном цикле.

**Варианты:**
- `left` / `right` — движутся навстречу друг другу (для палиндромов, поиска пар)
- `slow` / `fast` — медленный и быстрый указатели (для удаления дубликатов, перемещения нулей, фильтрации)

**Когда применять (вопросы-триггеры):**
- Работа со строкой или массивом?
- Есть симметрия (палиндром)?
- Нужно найти пару элементов?
- Массив отсортирован?
- Нужно сравнивать элементы с краёв?

**Код (JavaScript):**
```javascript
const isPalindrome = (s) => {
  let left = 0;           // Указатель на начало
  let right = s.length - 1; // Указатель на конец

  while (left < right) {  // Проход до середины строки
    if (s[left] !== s[right]) {
      return false;       // Несоответствие символов
    }
    left++;
    right--;
  }

  return true;
};

console.log(isPalindrome("madam"));  // true
console.log(isPalindrome("abc"));    // false
```

**Сложность (Big O):**
- ⏱ Time: `O(n)` — строка проходится один раз
- 💾 Space: `O(1)` — только две переменные-указателя

**Аналог LeetCode:** #125 — Valid Palindrome

### Паттерн 2: Hashmap (Хеш-таблица)

**Суть:** Структура `ключ → значение` (объект `{}` или `new Map()`). Позволяет избежать вложенных циклов за счёт сохранения встреченных элементов для мгновенного поиска, подсчёта частоты или группировки.

**Когда применять (вопросы-триггеры):**
- Нужно посчитать частоту элементов?
- Ищем уникальный / самый частый / анаграммы?
- Требуется быстрая проверка наличия без сортировки?
- Нужно сгруппировать элементы?

**Код (JavaScript):**
```javascript
const firstUniqueChar = (s) => {
  const map = {};  // HashMap для подсчёта частоты символов

  // Первый проход: подсчёт
  for (const char of s) {
    if (!map[char]) {
      map[char] = 1;
    } else {
      map[char] += 1;
    }
  }

  // Второй проход: поиск первого уникального
  for (const char of s) {
    if (map[char] === 1) {
      return char;
    }
  }

  return null;
};

console.log(firstUniqueChar("banana"));  // b
console.log(firstUniqueChar("aabbc"));   // c
```

**Сложность (Big O):**
- ⏱ Time: `O(n)` — два последовательных прохода по строке
- 💾 Space: `O(k)` — где `k` — количество уникальных символов (в худшем случае `O(n)`)

**Аналог LeetCode:** #387 — First Unique Character in a String

### Паттерн 3: Stack (Стек)

**Суть:** Структура `LIFO` (Last In — First Out). В JavaScript реализуется через массив методами `push()` и `pop()`. Применяется для проверки сбалансированности скобок, обработки вложенных структур, отката действий и хранения предыдущих состояний.

**Когда применять (вопросы-триггеры):**
- Есть скобки или вложенные структуры?
- Важен порядок закрытия / возврата?
- Нужно «откатиться назад»?
- Нужно помнить последний добавленный элемент?

**Код (JavaScript):**
```javascript
const isValidBrackets = (s) => {
  const stack = [];  // Стек для открывающих скобок
  const map = {
    ")": "(",
    "]": "[",
    "}": "{"
  };

  for (const char of s) {
    if (char === "(" || char === "[" || char === "{") {
      stack.push(char);  // Сохраняем открывающую скобку
    } else {
      const last = stack.pop();  // Извлекаем последнюю открытую
      if (last !== map[char]) {  // Проверка соответствия пар
        return false;
      }
    }
  }

  return stack.length === 0;  // Проверяем, остались ли незакрытые
};

console.log(isValidBrackets("{[()()]}"));  // true
console.log(isValidBrackets("(){}[]"));    // true
console.log(isValidBrackets("{[(])}"));    // false
```

**Сложность (Big O):**
- ⏱ Time: `O(n)` — один проход по строке, операции со стеком `O(1)`
- 💾 Space: `O(n)` — в худшем случае стек хранит все открывающие скобки

**Аналог LeetCode:** #20 — Valid Parentheses

## Чек-лист выбора паттерна

| Паттерн | Когда применять |
|:---|:---|
| **Two Pointers** | Строка/массив? Симметрия (палиндром)? Поиск пары? Отсортированный массив? Сравнение с краёв? |
| **Hashmap** | Подсчёт частоты? Уникальный/самый частый/анаграммы? Быстрая проверка наличия? Группировка? |
| **Stack** | Скобки/вложенные структуры? Порядок закрытия/возврата? Откат назад? Последний добавленный? |

## Как применять

### Шаг 1: Определи тип задачи

Задай себе вопросы из чек-листа. Если задача работает с:
- **Массивом/строкой + симметрия** → Two Pointers
- **Поиском/подсчётом/группировкой** → Hashmap
- **Скобками/вложенностью/откатом** → Stack

### Шаг 2: Выбери вариант паттерна

- Two Pointers: `left/right` для палиндромов, `slow/fast` для фильтрации
- Hashmap: объект `{}` для простого, `new Map()` для сложных ключей
- Stack: массив + `push/pop`

### Шаг 3: Реализуй решение

Следуй шаблону кода, адаптируя под конкретную задачу.

### Шаг 4: Проверь сложность

Убедись, что решение имеет O(n) Time и оптимальную空间 сложность.

## Ограничения и типичные ошибки

1. **Без практики паттерны не работают:** Чтение статьи не даёт навыка. Нужно решить 10-20 задач на каждый паттерн для выработки автоматического распознавания.

2. **Перепутывание вариантов:** `slow/fast` можно перепутать с `left/right`. Чётко понимай разницу: навстречу vs догоняние.

3. **Hashmap не всегда O(1):** В худшем случае (коллизии хешей) поиск может деградировать до O(n). В JavaScript это редкость, но стоит помнить.

4. **Stack требует проверки пустоты:** `stack.pop()` на пустом стеке вернёт `undefined`. Всегда проверяй `stack.length` перед `pop()` в критичных задачах.

5. **Two Pointers требует отсортированности:** Для задач поиска суммы пары (`Two Sum II`) массив должен быть отсортирован. Если не отсортирован — используй Hashmap.

6. **Не все задачи решаются этими паттернами:** Оставшиеся 40% требуют Dynamic Programming, Sliding Window, Binary Search, BFS/DFS и других.
