# История C#: Эволюция версий и экосистемы

## Что это такое

C# — один из самых популярных языков программирования, созданный Андресом Хейлсбергом (ранее — Turbo Pascal, Delphi) в Microsoft. Первая версия вышла в январе 2002 года вместе с .NET Framework.

Название имеет двойную трактовку:
- **C####** — C++ с двумя плюсами (в музыкальной нотации # — это диез)
- **До-диез** — музыкальная интерпретация

Язык вдохновлён C++, Java, Smalltalk, Delphi и Turbo Pascal.

## Почему это важно

Понимание эволюции C# помогает:
- **Правильно выбирать версии** для проектов (не все фичи доступны в старых .NET)
- **Понимать контекст решений** — почему язык развивался в сторону async/await, Span<T>, records
- **Осваивать новые возможности** постепенно, а не всё сразу
- **Оценивать экосистему** — Unity, Blazor, WPF, MAUI зависят от версий C#/.NET

## Ключевые вехи

### C# 1.0 (2002) — Старт

**Платформа:** .NET Framework 1.0, Visual Studio .NET 2002

**Ключевые возможности:**
- Классы, структуры, интерфейсы
- Делегаты и события
- Свойства (выглядят как поля, вызывают методы)
- Индексаторы (как свойства, но с параметром)
- Цикл `foreach`
- Перегрузка операторов
- Беззнаковые типы
- Оператор `goto`
- `unsafe`-код для работы с указателями
- Атрибуты
- Взаимодействие с неуправляемым кодом (DLL, COM)

**Цели проектирования:**
- Простой, современный, универсальный, ООП
- Строгая типизация, проверка границ массивов
- Автоматическая сборка мусора
- Переносимость исходного кода
- Поддержка интернационализации
- Применимость для основных и встраиваемых систем

### C# 2.0 (2005) — Обобщения

**Платформа:** .NET Framework 2.0, Visual Studio 2005

**Ключевые возможности:**
- **Generics (обобщённые типы)** — работают на уровне CLR (в отличие от C++ шаблонов)
- **Разделённые типы** (`partial`) — класс в нескольких файлах (полезно для source generators)
- **Анонимные методы** через `delegate` (замыкания)
- **Nullable-типы** (`int?`, `T?`)
- **Итераторы** и `yield return`
- Оператор `??`
- Отдельный `get`/`set` аксессоры
- Статические классы

### C# 3.0 (2007) — LINQ и функциональность

**Платформа:** .NET Framework 3.5, Visual Studio 2008

**Ключевые возможности:**
- **Автоматически реализуемые свойства** (`public int Age { get; set; }`)
- **Неявная типизация** (`var x = ...`)
- **Анонимные типы** (инкапсуляция свойств без явного объявления)
- **LINQ** (Language Integrated Query) — SQL-подобные запросы к коллекциям
- **Лямбда-выражения** (`(x, y) => x + y`)
- **Методы расширения** (добавление методов в существующие типы)
- **Инициализаторы объектов** (`new User { Name = "John" }`)
- Выражения как структуры данных (трансляция в SQL)

### C# 4.0 (2010) — Динамика и параллелизм

**Платформа:** .NET Framework 4.0, Visual Studio 2010

**Ключевые возможности:**
- **Динамические типы** (`dynamic`) — обход проверки типов на этапе компиляции
- **Именованные параметры** (`DoSomething(name: "value")`)
- **Опциональные параметры** (значения по умолчанию)
- **Контракты** — описания правил взаимодействия сущностей
- **TPL (Task Parallel Library)** — `Task`, `TaskFactory`, `Parallel`
- **`System.Collections.Concurrent`** — потокобезопасные коллекции (`ConcurrentQueue`)

### C# 5.0 (2012) — Асинхронность

**Платформа:** .NET Framework 4.5, Visual Studio 2012

**Ключевые возможности:**
- **`async` / `await`** — асинхронное программирование
- **Caller Info атрибуты** — получение контекста вызова без рефлексии

### C# 6.0 (2015) — Продуктивность

**Платформа:** .NET Framework 4.6, Visual Studio 2015

**Ключевые возможности:**
- **Null-условные операторы** (`?.`, `?[ ]`)
- **Импорт статических функций** (`using static Math`)
- **Условия в `catch`** (`when`)
- **`await` в `catch` / `finally`**
- **Инициализация автосвойств** (`public int Age { get; } = 25;`)
- **Expression-bodied функции** (`int Square(int x) => x * x;`)
- **Интерполяция строк** (`$"Hello, {name}!"`)
- **Оператор `nameof`** — получение имени типа в строковом формате
- **Инициализаторы словарей** (`new Dictionary<string, int> { ["key"] = 42 }`)
- **Компилятор Roslyn как сервис** (компилятор C# написан на C#)

### C# 7.0–7.3 (2017–2018) — Паттерны и производительность

**Платформа:** .NET Framework 4.7, Visual Studio 2017

**Ключевые возможности:**
- **`out` переменные** (`int.TryParse(s, out var result)`)
- **Кортежи значений** (`ValueTuple`) и деконструкторы
- **Паттерны** (`is` с шаблонами, `switch` с любыми типами)
- **`when` в `case`** — дополнительные условия
- **Локальные функции**
- **`ref` переменные и возврат** (`ref int Find(...)`)
- **`throw` в выражениях**
- **Бинарные литералы** (`0b1010_1010`)
- **Разделители в числах** (`1_000_000`)
- **Асинхронный `Main`** (`static async Task Main()`)
- **`default` литералы** (`default(T)`)

### C# 8.0 (2019) — Нулевая безопасность

**Платформа:** .NET Core 3.0

**Ключевые возможности:**
- **`readonly` методы** — гарантия неизменения состояния
- **Реализация по умолчанию в интерфейсах**
- **Рекурсивные шаблоны** — паттерн внутри паттерна
- **`switch` выражения** — сокращённый синтаксис
- **Шаблоны свойств и кортежей**
- **`using` объявления** — автоматическое освобождение `IDisposable`
- **`IAsyncEnumerable`** и `await foreach`
- **`IAsyncDisposable`**
- **Индексы и диапазоны** (`^` с конца, `..` диапазон)
- **Оператор `??=`** — присваивание при `null`
- **Неуправляемые сконструированные типы**

### C# 9.0 (2020) — Иммутабельность и простота

**Платформа:** .NET 5

**Ключевые возможности:**
- **`record` типы** — иммутабельные ссылочные типы (равенство по значениям, `with` для копий)
- **`init` аксессоры** — инициализация только при создании
- **Операторы верхнего уровня** — программа без `namespace`, `class`, `Main`
- **Шаблоны типов** (`is string s`)
- **Логические шаблоны** (`and`, `or`, `not`)
- **Реляционные шаблоны** (`> 0`, `<= 100`)
- **`new()` выражения** (с явным целевым типом)
- **`static` анонимные функции**
- **Нативные целые числа** (`nint`, `nuint`)
- **Указатели функций**

## Эволюция .NET

### .NET Framework (2002–2019)

- **2002:** .NET Framework 1.0 + CLR (Common Language Runtime)
- **2005:** .NET Framework 2.0 (generics)
- **2006:** .NET Framework 3.0 + WPF (Windows Presentation Foundation)
- **2008:** .NET Framework 3.5 + LINQ
- **2010:** .NET Framework 4.0 + TPL
- **2012:** .NET Framework 4.5 + async/await
- **2019:** .NET Framework 4.8 (последняя версия, поддержка прекращена)

### .NET Core → .NET (2016–настоящее время)

- **2016:** .NET Core 1.0 (кроссплатформенность, Open Source)
- **2017:** .NET Core 2.0
- **2018:** .NET Core 2.1 / 3.0 (Blazor Server, производительность)
- **2020:** **.NET 5** (переименование .NET Core → .NET, унификация)
- **2021:** .NET 6 (LTS)
- **2022:** .NET 7
- **2023:** .NET 8 (LTS, Blazor SSR, гибридные рендеринги)
- **2024:** .NET 9

## Связанные технологии

### Unity (2005)

Кроссплатформенный движок для разработки игр. Использует C# как основной язык. Поддерживает 25+ платформ (PC, консоли, мобильные, VR/AR). Модульная система компонентов.

### WPF (2006)

Фреймворк для desktop UI на Windows. Использует XAML для разметки. Отделение UI от логики. Открытый исходный код с 2018 года (MIT лицензия). Только Windows.

### Blazor (2018)

Фреймворк для веб-приложений на C# без JavaScript. Blazor Server (SignalR, серверное выполнение) и Blazor WebAssembly (клиентское выполнение через WASM). В .NET 8+ — смешанные режимы рендеринга.

### MAUI (.NET 6+)

Кроссплатформенный фреймворк для мобильных и desktop приложений. Наследник Xamarin.Forms. Единый код для iOS, Android, macOS, Windows.

## Ограничения и типичные ошибки

1. **Не все фичи доступны в старых .NET:** C# 8+ требует .NET Core 3.0+. C# 9+ требует .NET 5+. Проверяй совместимость.

2. **WPF только для Windows:** Не кроссплатформенный. Для кроссплатформы — Avalonia или Uno.

3. **Blazor WASM — декомпиляция:** Клиентские DLL можно декомпилировать. Не хранить секреты.

4. **`dynamic` — потеря типобезопасности:** `dynamic` обходит проверку типов на этапе компиляции. Используй только для COM-взаимодействия или JSON без схемы.

5. **`record` — не для мутабельных данных:** `record` иммутабелен по умолчанию. Для изменяемых данных используй класс.

6. **Версии .NET и C# — разные вещи:** C# 9 может использоваться в .NET 5, но не в .NET Framework 4.8. Проверяй таблицу совместимости.
