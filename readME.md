# База знаний по вайбкодингу
Obsidian-хранилище для сбора, обработки и систематизации знаний по AI-ассистированной разработке и вайбкодингу.

## Структура

| Директория | Назначение |
|---|---|
| [`raw/`](raw/readME.md) | Сырые материалы: статьи, заметки, скриншоты, транскрипты, PDF |
| [`Clippings/`](Clippings/) | Веб-вырезки, быстрые сохранения, фрагменты |
| [`wiki/`](wiki/readME.md) | Обработанные wiki-страницы — Map of Content: [wiki/readME.md](wiki/readME.md) |
| [`archive/`](archive/) | Архив исходников после обработки (хранение 90+ дней) |
| [`reports/`](reports/) | Отчёты по обработке и ревизии базы знаний |
| [`setup/`](setup/) | Конфигурации, шаблоны, скрипты инициализации |

## Принцип работы

1. **Приём** — всё входящее сначала попадает в `raw/` или `Clippings/`
2. **Классификация** — определение типа материала и темы
3. **Извлечение** — выделение ключевых идей, практических приёмов, ограничений
4. **Интеграция** — создание/обновление wiki-страниц в `wiki/`
5. **Навигация** — обновление `wiki/readME.md` как единой точки входа (MOC)
6. **Архивирование** — перенос обработанных оригиналов в `archive/`
7. **Отчётность** — сохранение отчётов в `reports/`

## Темы для развития

- AI IDE (Cursor, Windsurf, Claude Code)
- MCP (Model Context Protocol)
- Prompts / Agent Workflows
- Фреймворки и библиотеки
- Кейсы и анти-паттерны
- Монетизация и валидация ниш

## Правила

- Не смешивать сырые заметки и итоговые wiki-материалы
- Для новых тем — отдельные подпапки внутри `wiki/` с `readME.md` как точкой входа
- После каждого изменения wiki — актуализировать `wiki/readME.md` (MOC)
- Все технические термины и имена — в оригинале, пояснения — на русском
