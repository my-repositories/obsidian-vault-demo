## Краткое содержание

Статья представляет собой практическую шпаргалку по пяти принципам SOLID с акцентом на баланс между следованием принципам и здравым смыслом. Автор подчёркивает, что фанатичное применение SOLID вредит проектам, особенно MVP и high-load системам.

### Ключевые моменты

1. **SRP (Single Responsibility):** Один класс — одна причина для изменения
2. **OCP (Open/Closed):** Открыт для расширения, закрыт для изменения
3. **LSP (Liskov Substitution):** Наследник подменяет родителя без нарушения контракта
4. **ISP (Interface Segregation):** Маленькие специализированные интерфейсы вместо больших
5. **DIP (Dependency Inversion):** Зависимости от абстракций, не реализаций
6. **Ограничения:** Избыточность, падение производительности, конфликт с YAGNI/KISS
7. **Практики:** Code review, юнит-тесты, профилирование
