## Краткое содержание

Статья описывает типы `Span<T>` и `Memory<T>` в .NET, которые предоставляют типобезопасный способ работы с непрерывными областями памяти (стек, управляемая куча, неуправляемая память) без лишних аллокаций и копирований. Это критически снижает нагрузку на GC и повышает производительность.

### Ключевые моменты

1. **Три типа памяти в .NET:** стековая (`stackalloc`), управляемая (GC-куча), неуправляемая (`Marshal.AllocHGlobal`)
2. **Span<T>:** `ref struct` только для стека, слайсинг без копирования, чтение/запись
3. **Memory<T>:** обычная структура, работает с async/await, может храниться в куче
4. **ReadOnlySpan<T> / ReadOnlyMemory<T>:** аналоги для чтения
5. **Бенчмарк:** Span~7.5x быстрее Substring, 0 аллокаций Gen0
6. **Ограничения:** нельзя в поля классов, async/await, boxing, LINQ недоступен
7. **Альтернативы LINQ:** SpanLinq, NetFabric.Hyperlinq
8. **ReadOnlySequence<T>:** для несмежных буферов
