# Beads – Система памяти для кодинг-агентов

**Источник:** https://habr.com/ru/articles/983498/
**Дата сохранения:** 2026-04-12

## Полное содержимое статьи

### Основная проблема
Кодинг-агенты (Claude Code, Sourcegraph Amp, OpenAI Codex и др.) страдают от «деменции сессий»:
- Контекстное окно ограничено (~10 мин активной работы), после перезапуска память стирается.
- Агенты создают вложенные markdown-планы без связей между уровнями, теряют исходные фазы, рекурсивно плодят новые планы.
- При нехватке контекста агенты «срезают углы»: игнорируют сломанные тесты, отключают проверки, маркируют незавершённую работу как готовую.
- Обнаруженные в процессе баги не фиксируются, а игнорируются для экономии токенов.

### Техническая архитектура Beads (bd)
Beads (сокращение от bug database) — легковесный, агенто-ориентированный трекер задач/багов.
- Хранение: JSONL-файлы под управлением Git. Отказ от традиционных СУБД (PostgreSQL) и тяжелых оркестраторов (Temporal) в пользу контроля версий + возможности запросов. Конфликты слияния (в т.ч. коллизии ID на разных ветках) разрешаются ИИ через интеллектуальный merge.
- Схема данных: Минималистичная, но строго типизированная. Включает:
  - parent/child указатели (для эпиков и подзадач).
  - Блокирующие задачи (blocking).
  - 4 типа связей зависимостей, включая discovered-from (для работы, найденной во время выполнения другой задачи).
- Распределённость: Несколько агентов/воркеров на разных машинах работают с одной базой через общий Git-репозиторий.
- Формат данных: Повсеместное использование --json флагов для машинного чтения. Агенты не парсят текст, а запрашивают структурированные данные.

### CLI и ключевые команды
Инструмент управляется через утилиту bd:
- bd quickstart — инициализация и автоматический перенос открытых задач из markdown/TODO-листов в Beads.
- bd ready - json — запрос списка разблокированных задач, готовых к выполнению.
- --assignee <agent_id> — фильтрация задач по исполнителю для мульти-агентной координации.
- Статусы задач: open, in_progress, ready, blocked, closed.
- Аудит: Каждое изменение логируется с меткой времени и идентификатором агента. История статусов доступна без git blame.

### Практические рекомендации по внедрению
1. Отказ от markdown-планов: Перенести все текущие рабочие элементы в Beads. Markdown подходит только для документации, не для оркестрации.
2. Декомпозиция задач: Дробить работу на мелкие bd-тикеты. Каждая сессия агента выполняет ровно одну задачу. Это:
   - Снижает стоимость вычислений (квадратичная экономия токенов).
   - Убирает паническое «закрытие» задач при приближении лимита контекста.
   - Делает сессии одноразовыми и предсказуемыми.
3. Интеграция в промпты: Добавить в AGENTS.md или CLAUDE.md одну строку-инструкцию: использовать bd quickstart и автоматически заводить тикеты на любые обнаруженные проблемы (включая сломанные тесты или техдолг).
4. Мульти-агентная работа: Агенты перед стартом проверяют status: in_progress через bd, что исключает дублирование. Разные ветки и машины синхронизируются через Git.
5. Настройка: Установка утилиты bd → запуск bd quickstart против репозитория → агенты автоматически переключаются на тикетную модель.

### Архитектурные уроки и ограничения
- Temporal избыточен для легковесных десктопных инструментов оркестрации агентов.
- Markdown-планы не масштабируются для долгосрочного планирования ИИ. Ведут к фрагментации и потере контекста.
- TypeScript не рекомендуется для генерации ИИ-оркестраторов: строгая типизация не мешает ИИ писать посредственный код, а ошибки архитектуры проявляются поздно. Автор переписывает свой проект vibecoder на Go.
- Beads находится в стадии alpha, но уже демонстрирует стабильную работу в production-сценариях вайб-кодинга.

### Оценка эффективности со стороны ИИ (Claude)
- Markdown для агентов — память только для записи (write-only). Требует ручного копирования контекста или парсинга текста.
- В Beads зависимости — first-class citizens. Запрос bd ready - json мгновенно выдаёт пул доступных задач без интерпретации прозы.
- Граф зависимостей отражает реальный ход разработки, а не статичный список. Поле discovered-from связывает баги с задачами, в которых они были найдены.
- Возврат к markdown после Beads сравнивается с попыткой удержать длинный телефонный номер в памяти без записи.
- Система спроектирована под ограничения агентов: лимиты контекста, отсутствие постоянной памяти, потребность в структурированных данных и явной семантике.
